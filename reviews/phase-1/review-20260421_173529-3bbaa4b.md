#### MEDIUM — Error responses are cached unconditionally in `score_tickers_with_quant`

**File:** `tradingagents/graph/prefilter.py:147`  
**Issue:**

```python
_save_cached_raw(cache_path, symbol, trade_date, quant_kwargs, parsed)
```

This write occurs before the error check. If the quant tool returns `{"error": "No price history returned"}` (a transient network failure, weekend date, or data-vendor gap), that error payload is persisted to cache and will be served on every subsequent call until TTL expiry or `refresh_cache=True`.  
**Impact:** A transient fetch failure silently poisons the cache for `cache_ttl_days` days. The ticker will score as `-inf` and be excluded from selection for the full TTL window without any visible signal to the operator.  
**Minimal fix:** Skip the cache write when the parsed payload contains an `"error"` key:

```python
if not parsed.get("error"):
    _save_cached_raw(cache_path, symbol, trade_date, quant_kwargs, parsed)
```

#### MEDIUM — Handoff describes 80-bit cache key but implementation truncates to 64 bits

**Files:** `tradingagents/dataflows/intraday.py:71`, `tradingagents/graph/prefilter.py:33`  
**Issue:** Both `_cache_key` in `intraday.py` and `_get_cache_path` in `prefilter.py` use `hexdigest()[:16]` (64-bit prefix). The handoff review checklist says "SHA-256 truncation to 20 hex chars (80 bits)". At 64 bits the collision probability is ~N²/2^65 — negligible for practical universe sizes (≤10,000 tickers) — but the spec/implementation mismatch is a documentation debt that will confuse Phase 2 authors who read the handoff.  
**Impact:** No practical collision risk, but the documented contract is wrong.  
**Minimal fix:** Update the handoff "What the reviewer must focus on" entry to say "16 hex chars (64 bits)" and optionally add a comment in both files noting the bit-length choice.
