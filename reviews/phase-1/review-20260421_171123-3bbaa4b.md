#### HIGH — TOCTOU race in `get_intraday_bars` can persist live data to historical cache

**File**: `tradingagents/dataflows/intraday.py`, lines 339 and 353

**Issue**: `_is_live_end_date(end)` is called twice without storing the result. If UTC midnight falls between the two calls, the first call returns `True` (live — skip cache read) and the second returns `False` (now historical — save to cache). This writes partial live-day data into the historical parquet cache under the historical key.

```python
# line 339
use_cache = (not refresh_cache) and (not _is_live_end_date(end))  # may see True
...
# line 353
if not _is_live_end_date(end):     # may see False seconds later
    _save_cache(path, df)          # partial live data cached as historical
```

**Impact**: On the next call with the same key, `use_cache=True` loads the stale partial-day DataFrame. All downstream Phase 2 callers (regime classifier, entry engine) receive a truncated bar history with no error raised. The cache persists until `refresh_cache=True` is explicitly set — the system does not self-heal. This violates the "no stale bar data" requirement directly.

**Minimal fix** (2 lines):

```python
is_live = _is_live_end_date(end)            # capture once
use_cache = (not refresh_cache) and (not is_live)
if use_cache:
    df = _load_cache(path)
if df is None:
    df = fetch_intraday_bars(...)
    if not is_live:                          # use captured value
        _save_cache(path, df)
```

#### MEDIUM — `_align_session` regular-session open mask drops the 09:00–10:00 ET 1h bar used in 4h resampling

**File**: `tradingagents/dataflows/intraday.py`, line 141

**Issue**: The open-boundary mask is:

```python
hour_mask = (idx_et.hour > 9) | ((idx_et.hour == 9) & (idx_et.minute >= 30))
```

yfinance 1h bars are labeled at the top of each hour. The bar covering 09:00–10:00 ET has `hour=9, minute=0`, which fails the mask and is filtered. For the 4h resampling path, this means the first NYSE 1h bar of the day (09:00–10:00 ET, which includes the 09:30 open auction) is silently dropped. The first included 1h bar is 10:00 ET.

**Impact**: 4h candles in Phase 2 will never contain the 09:30 opening bar. Breakout and momentum strategies that depend on the opening range will be biased. The 4h candle labelled `12:00 UTC` (EST example) represents 10:00–14:00 ET instead of 09:30–13:30 ET. Phase 2 must be told this explicitly; it is not documented in the handoff or the module docstring.

**Minimal fix**: Add a note to the `_align_session` docstring and to the Phase 1 handoff under "Known limitations":

> "For 4h bars via yfinance, the 09:00 ET 1h bar (covering 09:30 open) is excluded by the session mask because yfinance labels 1h bars at the top of the hour. The first included 1h bar is 10:00 ET. Phase 2 callers must not rely on 4h candles capturing the 09:30 open."

A code fix (extending the mask to include the 09:00 bar when the bar covers the open) would need to be evaluated against whether yfinance's 09:00 bar reliably includes 09:30 volume.

#### LOW — `_is_live_end_date` naive date comparison is only safe by coincidence

**File**: `tradingagents/dataflows/intraday.py`, line 117–120

**Issue**:

```python
end_date = pd.Timestamp(end).date()      # naive date (no tz)
return end_date >= datetime.now(timezone.utc).date()  # utc date
```

`pd.Timestamp("2026-04-21").date()` returns `datetime.date(2026, 4, 21)` with no timezone. `datetime.now(timezone.utc).date()` also returns a bare `datetime.date`. Python date objects have no tz, so the comparison is safe in isolation, but the semantics are ambiguous: the `end` string is compared as-if it is a UTC calendar date. If a caller passes a date string that is local time and local time is behind UTC, a date that is "tomorrow" in local time (but today in UTC) would incorrectly be treated as non-live.

**Impact**: Low — only affects callers in UTC-ahead timezones where `end` is tomorrow local time. No impact on US market contexts. No fix required for Phase 1; document the assumption.

### Positive observations

- `_enforce_no_lookahead` correctly uses `df.index < cutoff` (strict less-than), confirmed by test. The bar at exactly `as_of` is excluded.
- `_align_session` delegates DST handling to `pytz.timezone("America/New_York")` via `tz_convert`, which is correct. The DST spring-forward and fall-back tests are accurate and cover real transition dates.
- Regular session boundary at 16:00 ET is correctly exclusive (`idx_et.hour < 16`); bars at 15:45 survive, bars at 16:00 are dropped.
- Cache key truncation: intraday uses 20 hex chars (80 bits), prefilter uses 16 hex chars (64 bits). Both are collision-safe for realistic ticker/date volumes.
- `as_of` is intentionally excluded from the intraday cache key — the full date range is stored and `as_of` trimming is applied post-load. This is correct and well-documented.
- `interface.get_intraday_bars` correctly bypasses `route_to_vendor` (which has alpha_vantage fallback logic that would fail silently for a vendor with no intraday implementation). The direct dispatch with an explicit `ValueError` on unknown vendor is the right approach.
- `_resample_1h_to_4h` OHLCV aggregation: open=first, high=max, low=min, close=last, volume=sum — all correct. Partial window filtering via `counts >= 2` correctly prevents single-bar candle artifacts.
- Session filter applied before resampling (correct order — prevents cross-session aggregation in 4h candles).
- `vectorbt` is correctly in `[project.optional-dependencies]` (not hard deps), consistent with the try/except guard in `quant_tools.py`. `pyarrow` is correctly a hard dependency.
- TTL cache expiry in prefilter uses explicit UTC timestamps with `Z`/`+00:00` normalization — handles older cache files without crashing.

### Merge decision: CHANGES_REQUIRED

**Top 3 required fixes:**

1. **(HIGH)** `intraday.py`: capture `_is_live_end_date(end)` result in a single variable and use it for both the cache-read guard and the cache-write guard. Eliminates the TOCTOU race that can persist partial live-day data to the historical cache.

2. **(MEDIUM)** `prefilter.py` + handoff: add at least one test assertion that exercises the `QuantSignalContract` layer (e.g., check `result["ranked"][0]["signal"] == "buy"`). Update the handoff to reflect that `QuantSignalContract.from_raw()` is the normalization path actually used.

3. **(MEDIUM)** `intraday.py` docstring + Phase 1 handoff: document that the 09:00 ET 1h bar is excluded by the session mask, so 4h candles do not capture the NYSE 09:30 open. Phase 2 must not assume the first 4h candle of the day includes the opening bar.
