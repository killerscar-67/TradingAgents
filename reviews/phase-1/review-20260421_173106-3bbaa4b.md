**Impact**: All 13 tests in `test_intraday.py` are unreachable. The handoff test command `tradingagent_venv/bin/python -m unittest tests.test_intraday ...` exits with an error on import, giving a false "33 tests, all OK" claim.

#### MEDIUM — `interface.get_intraday_bars` does not forward `vendor` to the implementation

**File**: `tradingagents/dataflows/interface.py:203–211`  
**Issue**: The interface resolves `vendor` from config/tool_vendors, looks up `impl = VENDOR_METHODS["get_intraday_bars"].get(vendor)`, but then calls `impl(...)` **without** passing `vendor=` as a keyword argument. `intraday.get_intraday_bars` has `vendor="yfinance"` as its default, so the resolved vendor in the interface and the vendor used inside the implementation are independently set to the same value today — but only by coincidence.

```python
impl = VENDOR_METHODS["get_intraday_bars"].get(vendor)  # vendor selected here
...
return impl(
    symbol=symbol, interval=interval, start=start, end=end,
    as_of=as_of, session=resolved_session,
    cache_dir=resolved_cache_dir, refresh_cache=resolved_refresh_cache,
    # vendor= NOT forwarded — impl defaults to "yfinance" independently
)
```

**Impact**: Adding a second vendor to `VENDOR_METHODS["get_intraday_bars"]` and mapping it to `intraday.get_intraday_bars` (the same underlying function) would cause the implementation to silently use yfinance regardless of the resolved vendor. This is a latent maintenance trap.

**Minimal fix**: Add `vendor=vendor` to the `impl(...)` call.

#### LOW — Prefilter cache digest length inconsistency (64-bit vs 80-bit)

**Files**: `tradingagents/graph/prefilter.py:31` vs `tradingagents/dataflows/intraday.py:71`  
**Issue**: Prefilter uses `hexdigest()[:16]` (64 bits); intraday uses `hexdigest()[:20]` (80 bits). Neither length is a collision risk for realistic inputs, but the inconsistency is unexplained and makes auditing harder.

**Impact**: None in practice; purely a consistency concern.

**Minimal fix**: Standardise both to `[:20]` (80 bits) — the longer value already chosen by the intraday module.

#### LOW — Handoff "at most 1 usable NYSE-session candle per day" for 4h is inaccurate

**File**: `docs/handoffs/phase-1.md` (Known Limitations)  
**Issue**: For a standard NYSE regular session, `_align_session` retains six 1h bars (14:00–19:00 UTC in EST, 13:00–18:00 UTC in EDT). UTC-midnight 4h resampling (`resample("4h")`) produces two buckets with ≥ 2 bars each, yielding **two** 4h candles per day, not one. The documented "at most 1" is under-counting.

**Impact**: Documentation only; does not affect code correctness. Phase 2 callers should not rely on the "1 candle" count.

**Minimal fix**: Update Known Limitations to read "at most 2 usable NYSE-session 4h candles per day (one partial boundary candle dropped when count < 2)."

### Session/Timezone Correctness

The `_align_session` implementation is correct:

- **Regular session open boundary**: `(hour > 9) | (hour == 9 and minute >= 30)` — includes the 09:30 ET bar exactly as required; pre-market 09:15 ET excluded. ✓
- **Regular session close boundary**: `hour < 16` — excludes the bar labeled at 16:00 ET exactly. ✓
- **DST spring-forward (2026-03-09)**: pytz `America/New_York` handles the 1h clock advance transparently; test coverage confirms 09:30 ET = 13:30 UTC is included. ✓
- **DST fall-back (2026-11-02)**: test coverage confirms 09:30 ET = 14:30 UTC is included. ✓
- **Extended session**: `hour >= 4 and hour < 20 ET` on weekdays — correct.
- **Crypto**: unfiltered — correct.

### No-Lookahead Guarantee

`_enforce_no_lookahead` uses strict `df.index < cutoff` (not `<=`), meaning bars at exactly `as_of` are excluded. This is the correct semantics for a strategy clock: the bar whose open **equals** the current time is not yet closed, and must not be visible. Test at line 82–86 covers this explicitly. ✓

`get_intraday_bars` captures `is_live = _is_live_end_date(end)` once (TOCTOU fix from Round 5) and uses it for both the cache-read guard and the cache-write guard. ✓

### Deterministic Cache Behavior

- Cache key covers `(v, symbol.upper(), interval, start, end, session)` using `sort_keys=True` JSON — deterministic across Python implementations. ✓
- `as_of` is correctly excluded from the key; trimming is applied post-load so the stored payload is reusable across different `as_of` values. ✓
- 20-hex SHA-256 (80-bit) is collision-safe for any realistic ticker/date universe. ✓
- Live-end-date guard prevents writing stale intraday snapshots for open ranges. ✓
