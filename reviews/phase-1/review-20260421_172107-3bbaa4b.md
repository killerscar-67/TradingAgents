#### MEDIUM — Stale mock side_effect in TOCTOU regression test

**File:** `tests/test_intraday.py:269`

**Issue:** After Round 5 fixed the TOCTOU race by capturing `_is_live_end_date(end)` into a single `is_live` local variable, the function is called exactly **once** per `get_intraday_bars` invocation. But `test_live_end_date_fetch_is_not_persisted_to_cache` still uses `side_effect=[True, True, False, False]` (4 values), a leftover from the pre-TOCTOU state where it was called twice per invocation. Two extra values are silently consumed by the Mock and never verified.

**Impact:** The test passes, but it does not guard the TOCTOU fix. If Round 5 were reverted (restoring two calls per invocation), the mock would still return `True, True` for both calls inside the first `get_intraday_bars` call, correctly suppressing the cache write — the test would still pass. The regression test is therefore inert against the specific bug it is meant to prevent.

**Minimal fix:**
```python
# tests/test_intraday.py:269
with patch(
    "tradingagents.dataflows.intraday._is_live_end_date",
    side_effect=[True, False],   # one value per get_intraday_bars call
) as mock_is_live:
    ...

self.assertEqual(mock_fetch.call_count, 2)
self.assertEqual(mock_is_live.call_count, 2)   # add: verify single-call invariant
```

#### LOW — `_is_live_end_date` naive date comparison is undocumented at call sites

**File:** `tradingagents/dataflows/intraday.py:114-120`

**Issue:** `pd.Timestamp(end).date()` produces a naive date, compared to `datetime.now(timezone.utc).date()`. Both strip timezone info. For callers in UTC+12..+14 timezones who pass a local date string as `end`, their "today" may already be UTC "tomorrow", causing `_is_live_end_date` to return `False` when it should return `True`, potentially writing stale intraday data to cache.

**Impact:** Incorrect live/historical classification for UTC-ahead callers passing local dates. The handoff documents this as a known limitation but neither `get_intraday_bars` nor `interface.get_intraday_bars` docstrings warn callers.

**Minimal fix:** Add one line to the `get_intraday_bars` docstring: "`end` must be a UTC calendar date string. Local dates in UTC-ahead timezones may trigger unexpected cache writes."
