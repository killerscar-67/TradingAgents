# Phase 12 — Review fix notes (2026-04-24)

Review: [reviews/phase-12/review-20260423_182016-a0a6585.md](../../../../reviews/phase-12/review-20260423_182016-a0a6585.md)

## Items resolved

### 🔴 HIGH — POST `/api/backtests` missing `strategy_id`, `start_date`, `end_date`

**Fix**: `tradingagents/web/routes/workflow.py` `create_backtest()` now includes
`"strategy_id": backtest.get("strategy_id")`, `"start_date": backtest.get("start_date")`,
and `"end_date": backtest.get("end_date")` in the return dict.  
The frontend `BacktestApiResponse` interface is now satisfied by the real backend response.

### 🔴 HIGH — `GET /api/backtests/{backtest_id}` was dead code

**Fix**: `BacktestScreen.tsx` re-fetch `useEffect` guard changed from
`run?.backtest_id === backtestId` to unconditional — the effect now fires whenever
`done && backtestId` regardless of whether a run is already set from the POST response.
The GET endpoint is therefore always called after terminal SSE, hydrating the final
persisted record. A new `BacktestScreen.test.tsx` test drives terminal SSE and asserts
the follow-up `GET /api/backtests/bt-001` is called and the KPIs from the GET response
are rendered.

### 🟡 MEDIUM — `BatchScreen` dropped all but the last SSE event per render cycle

**Fix**: `BatchScreen.tsx` now uses a `useRef(lastProcessedEventIndex)` cursor.
The effect slices `events.slice(lastProcessedEventIndex.current)`, processes every
pending event, then advances the cursor to `events.length`. A new
`BatchScreen.test.tsx` burst-event test emits two simultaneous `batch_item` events
and asserts both symbol result cards show `completed`.

### 🟡 MEDIUM — No dedicated test for `GET /api/backtests/{backtest_id}`

**Fix**: `tests/test_web_workflow_contracts.py` now includes
`test_get_backtest_detail_returns_stored_record_and_404_for_unknown` which:
- seeds a backtest via `POST /api/backtests`,
- calls `GET /api/backtests/{backtest_id}` and asserts the stored record shape
  (`backtest_id`, `strategy_id`, `start_date`, `end_date`, `status`, `result`),
- calls `GET /api/backtests/nonexistent` and asserts HTTP 404.

### 🟢 LOW — `StrategyScreen` Futu test did not assert `orders` shape

**Fix**: `StrategyScreen.test.tsx` now parses the Futu POST body and asserts
`orders[0]` matches `{ symbol: "AAPL", side: "buy", quantity: 50, entry_price: 200 }`.
